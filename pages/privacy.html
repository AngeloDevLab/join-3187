2. Overview of Processing

This app processes personal data to provide a Kanban board (JOIN). No cookies, tracking, or analytics tools are used.

3. Registration

During registration we collect:

  • Name
  • Email address

This data is permanently stored in the Firebase Realtime Database (collection users). The entered password is checked client-side only and is never transmitted to or stored on the backend.

Legal basis: Art. 6(1)(b) GDPR (performance of a contract / pre-contractual measures).

4. Login

The email address is used for identification during login. After a successful login, the following data is stored in the browser's sessionStorage:

  • ID, name, initials, email address

This data is automatically deleted when the browser tab is closed. This is not a cookie.

Guest login: No data is stored in the database; all information remains local in sessionStorage.

Legal basis: Art. 6(1)(b) GDPR.

5. Use of the App (Contacts and Tasks)

While using the app, the following data is stored in the Firebase Realtime Database:

Contacts (collection contacts): Name, email address, phone number

Tasks (collection tasks): Title, description, due date, priority, category, subtasks, assigned contacts

This data is visible to all users of the app (no per-user data separation).

Legal basis: Art. 6(1)(b) GDPR.

6. Hosting and Processing by Third Parties

We use Firebase Realtime Database by Google to store all data mentioned above.

Provider: Google Ireland Limited, Gordon House, Barrow Street, Dublin 4, Ireland

It cannot be ruled out that data is also processed on servers outside the EU. Google uses EU Standard Contractual Clauses as a safeguard for an adequate level of data protection.

Database location: Belgium (europe-west1).

More information: Firebase Privacy Policy by Google.

7. No Cookies, No Tracking

This app does not use cookies, analytics tools (e.g. Google Analytics), or tracking. Fonts and icons are loaded locally; there is no connection to external CDNs (e.g. Google Fonts).

8. Storage Duration

Data in the Firebase Realtime Database (users, contacts, tasks) is stored until manually deleted. There is no automatic deletion or retention period.

10. Data Security

Data transmission is encrypted via HTTPS.

The Firebase Realtime Database is not secured by Firebase Security Rules based on user identity. All app data (contacts, tasks) is readable and writable by any user of the app, including guests, without server-side access control differentiating between individual accounts. This is a deliberate design decision for this shared project management environment.
